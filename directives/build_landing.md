# 홈 랜딩페이지 구축 지침 (build_landing.md)

## 목표
용병 서비스의 홈 랜딩페이지(`pages/index.html`)를 구축한다.

## 입력
- agent.md (서비스 개요, 디자인 방향, 페이지 스펙)

## 도구/스크립트
- `execution/build_pages.py`

## 출력
- `pages/index.html` (완성된 HTML 단일 파일)

## 페이지 구성

### 1. Hero 섹션
- 헤드라인: "세트가 무너지기 전에, 용병을 부르세요"
- 서브카피: "숙련된 배달 라이더가 빠르게 출동합니다"
- CTA 버튼 2개:
  - Primary (레드 채움): `용병 라이더로 가입` → `/register`
  - Secondary (아웃라인): `긴급 라이더 요청` → `/register`

### 2. 작동 방식 섹션 (How It Works)
- 지사 흐름: 요청 등록 → 용병 매칭 → 출동 → 세트 사수
- 라이더 흐름: 프로필 등록 → 요청 확인 → 지원 → 출동 → 수익

### 3. 숫자로 보는 플랫폼
- 등록 용병 수 (임시값)
- 매칭 완료 건수 (임시값)
- 세트 사수율 (임시값)

### 4. 요금제 섹션 (티저)
- 지사용 요금제 요약 (3가지 플랜 카드)
- "자세히 보기" → `pricing.html`

### 5. 하단 CTA 섹션
- "지금 바로 시작하세요"

## 디자인 방향
- 다크 배경: `#0a0a0a` ~ `#1a1a1a`
- 포인트 컬러: `#E53935` (레드)
- 서브: `#ffffff`, `#888888`
- 풀스크린 히어로, 다크 카드, 레드 배지/아이콘
- 반응형: 375px ~ 1440px

## 엣지 케이스
- 히어로 배경 이미지 없으면 CSS gradient fallback 사용
- 모든 숫자는 임시값, 코드 상단 주석으로 분리

## 완료 조건
- HTML 문법 오류 없음
- 모바일/데스크탑 반응형 확인
- 모든 링크 상대경로로 연결
