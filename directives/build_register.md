# 회원가입 페이지 구축 지침 (build_register.md)

## 목표
용병 서비스의 회원가입 페이지(`pages/register.html`)를 구축한다.

## 입력
- agent.md (회원가입 스펙)

## 출력
- `pages/register.html`

## 페이지 구성

### 1단계: 가입 유형 선택 화면
- "용병 라이더로 가입" 카드 버튼
- "플러스 지사로 가입" 카드 버튼

### 2단계-A: 용병 라이더 가입 폼
| 필드 | 타입 |
|------|------|
| 이름 | text |
| 연락처 | tel |
| 활동 지역 | checkbox (복수 선택) |
| 배달 경력 | radio (1년 미만 / 1~3년 / 3년 이상) |
| 이메일 | email |
| 비밀번호 | password |
| 비밀번호 확인 | password |

### 2단계-B: 플러스 지사 가입 폼
| 필드 | 타입 |
|------|------|
| 지사명 | text |
| 사업자등록번호 | text |
| 담당자 이름 | text |
| 연락처 | tel |
| 이메일 | email |
| 비밀번호 | password |
| 비밀번호 확인 | password |

## UX 규칙
- JS로 유형 선택 → 해당 폼만 표시 (단일 파일 내 탭 전환)
- 비밀번호 확인 불일치 시 인라인 오류 표시
- 가입 완료 후 로그인 페이지로 이동 (현재 MVP: alert + redirect)

## 디자인
- 홈 랜딩페이지와 동일한 다크 테마
- 입력 필드: 다크 배경 + 레드 포커스 테두리

## 완료 조건
- HTML 문법 오류 없음
- 반응형 확인
